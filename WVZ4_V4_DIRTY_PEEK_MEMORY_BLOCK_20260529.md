# WVZ4 v4 Dirty Peek Memory Block Optimization (20260529)

This change adds a conservative memory-block precheck for dirty peek groups.

## What changed

- Added dirty-peek memory-block grouping under each dirty peek group.
- Adjacent dirty peek scalar leaves are grouped by address when the inter-leaf gap is no more than `dirty_peek_memory_block_max_gap` bytes. The default is `7`.
- Mixed scalar types are allowed in the same memory block. The block is used only as a raw changed-byte precheck; actual event emission still reads each leaf using its normal typed sampling path.
- SIMD changed-byte scanning reuses the existing SSE2/AVX2 backend selection.
- Leaves that cannot be safely represented as direct memory bytes, or that do not meet block thresholds, fall back to the original per-leaf dirty peek path.

## New BuildOptions

```cpp
bool enable_dirty_peek_memory_block_precheck = true;
std::size_t dirty_peek_memory_block_max_gap = 7;
std::size_t dirty_peek_memory_block_min_leaf_count = 8;
std::size_t dirty_peek_memory_block_max_bytes = 4096;
bool enable_dirty_peek_memory_block_simd_mask = true;
std::size_t dirty_peek_memory_block_simd_min_bytes = 64;
```

## Safety model

- Gaps are allowed, but changed bytes inside gaps only cause a conservative scan decision; no event is emitted unless a leaf byte range overlaps the changed bytes.
- Mixed types are allowed because final events are generated by precise leaf sampling, not by reinterpreting the memory block.
- The feature assumes the existing WaveTap sampling contract: business threads must finish the cycle before sampling.
